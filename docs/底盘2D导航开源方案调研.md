# Helios 轮臂机器人底盘 2D 导航开源方案

适用对象：珞石 Helios 轮式人形机器人移动底盘
目标：脱离厂家 MATRIX 界面，在公司统一平台上用开源算法实现底盘 2D 建图与导航
环境：ROS2 Humble、Orin 算力机（192.168.71.51）、底盘（192.168.71.50）、全向移动底盘、前后双 2D 激光雷达

---

## 一、需求背景

1. 厂家 MATRIX 只能在其自带界面完成建图与导航，无法纳入公司平台统一管理。
2. 公司希望实现多机器人统一管理与调度，需要可控的导航算法栈。
3. 已验证可从底盘获取雷达数据接口（详见《雷达测试》文档），具备自建导航的基础。

---

## 二、现状与接口情况

### 2.1 已确认可获取的接口

| 接口 | 形式 | 状态 |
|---|---|---|
| 前雷达 | `/sr_amr_control/front/scan`（sensor_msgs/LaserScan） | 已验证 |
| 后雷达 | `/sr_amr_control/rear/scan`（sensor_msgs/LaserScan） | 已验证 |
| 速度控制 | `/sr_amr_control/remote_control_cmd_vel`（geometry_msgs/Twist） | 需先开启远程控制 |
| 系统状态 | `/sr_amr_control/system_state` | 已验证 |
| 电池状态 | `/sr_amr_control/battery_state` | 手册提供 |

### 2.2 待确认接口（开发前置条件）

| 接口 | 重要性 | 当前状态 |
|---|---|---|
| 里程计 odom | 必需（amcl 依赖） | 需在 Orin 实测确认 |
| 真实 TF 外参（base_link↔雷达） | 必需 | 测试中临时用 `frame_id:=map`，需补真实外参 |
| IMU | 可选（用于后续提升定位稳定性） | 需确认 |

> 核实方式：在 Orin 上执行 `ros2 topic list -t`、`ros2 run tf2_tools view_frames`，并查阅 SDK 自带 `README-zh.md`。

### 2.3 雷达数据已知限制

- 频率约 5Hz，角分辨率 1°（721 个角度 bin）。
- 无效值用 `101.0` 占位，需转换为 `inf` 才能被 SLAM/Nav2 正确处理。
- 为 SROS 处理后的障碍重建扫描，非原始 ring data；仅发布 LaserScan。

## 三、推荐方案

采用 ROS2 Nav2 生态，技术栈如下：

```
ROS2 Humble
├── slam_toolbox    2D 激光建图
├── map_server      地图加载 / 保存 / 发布
├── amcl            基于地图的定位
└── nav2            路径规划 + 避障 + 全向局部控制
```

链路：

```
slam_toolbox 建图
    → map_server 保存并发布地图
        → amcl 在地图上定位
            → nav2 规划路径、避障、导航到目标
                → /cmd_vel 下发底盘
```

各组件职责：

| 组件 | 职责 |
|---|---|
| slam_toolbox | 激光建图，输出栅格地图 |
| map_server | 管理地图，向定位与导航发布地图 |
| amcl | 在已知地图上做粒子滤波定位，输出 map→odom |
| nav2 | 全局规划、局部避障、轨迹跟踪；全向底盘局部控制器配置 MPPI（Omni 模型） |

平台与算法分工：

- 公司平台：任务下发、目标点 / 站点、地图管理、状态监控、调度。
- 机器人本体：运行上述算法栈，负责实时定位、避障、路径跟踪等安全相关功能。

---

## 四、定位选型说明

- **当前方案：amcl。** 成熟稳定、Nav2 标配、资料完善；对当前偏低质量雷达更宽容，与工作安排一致。前提是底盘可提供可用 odom 和 TF。
- **后续可选项：slam_toolbox localization。** 在完成基础导航后，可作为升级选项引入，其扫描匹配对 odom 依赖更低、内置重定位能力，适合 odom 较弱或需要边定位边更新地图的场景。

---

## 五、落地阶段规划

| 阶段 | 内容 | 产出 |
|---|---|---|
| 1. 接口核实 | 确认 odom / TF（及 IMU）是否可获取 | 接口核查结论 |
| 2. 数据预处理 | 雷达 `101.0→inf` 过滤、补真实 TF 外参、双雷达处理 | 可用 scan + TF |
| 3. 建图 | slam_toolbox 实跑建图，map_server 保存地图 | 地图文件 |
| 4. 定位 | amcl 在地图上定位，验证位姿准确性 | 稳定定位 |
| 5. 导航 | nav2 低速发目标点，到点 + 避障 | 闭环导航 demo |
| 6. 平台对接 | 封装任务 / 状态 API，接入公司平台 | 平台可下发任务 |

---

## 六、风险与注意事项

1. odom / TF 若厂家未开放，需向厂家申请或基于编码器自建里程计，否则 amcl 定位无法运行。
2. 雷达 5Hz / 1° 偏低，需实测移动状态下的建图与避障效果。
3. 轮臂机器人特殊适配：
   - 导航时双臂应回收至安全姿态；
   - nav2 footprint 按机器人最大外形配置，而非仅底盘尺寸；
   - 初期速度限制在 0.2~0.4 m/s；
   - 2D 激光存在盲区，建议后续接入 TOF 补低矮 / 盲区障碍。

---

## 七、结论

- 雷达接口已验证可用，具备自建导航基础。
- 推荐方案：**slam_toolbox + map_server + amcl + nav2**，定位采用 amcl，slam_toolbox localization 作为后续升级可选项。
- 开发前置条件：确认 **odom / TF** 可获取；若暂不可得，需先解决里程计来源。
